## Open questions / 監査 limitation

1. **「decl 名対応の plan 参照」基準が plan ごとに緩い**: 一部 plan (closure, moonshot) は
   Phase 名と概念 (PL n-dim Fubini, prob↔幾何 bridge 等) で記述、code 上の declaration 名と
   1-to-1 対応していない。本 audit は「declaration 名が plan 本文に literal 出現するか」で
   in-plan orphan を判定したため、概念対応している項目も orphan 計上している可能性あり (
   特に Concavity wrapper / Functional の `rpow_*`)。

2. **`EPIL3Integration.lean` 19 件は plan §Phase E の補助 corollary 群との対応 mapping が
   不明確**: 名前 prefix (`*_integrated` / `*_reduced` / `*_full`) は plan §Phase E の `multi-arg,
   monotonicity, scaling, log-form` と意味的に対応するが、plan 本文での明示参照なし。本 audit
   では in-plan orphan として計上したが、plan 側に file 名・declaration 名を追記すれば 19 件分の
   orphan が解消する (close ROI medium 群)。

3. **`AWGNAchievabilityDischarge.lean:32,117` の prose 言及 2 件**: declaration tag ではないが
   `rg @audit:suspect(differential-entropy-plan)` で hit する。本 audit では実 declaration tag
   と区別したが、grep ベースの自動集計 (e.g. `defect-inventory` script) では 6 件として表示
   されるので、count の同期に注意。

4. **本 audit はコード上の `@audit:suspect` tag を SoT とする** ([feedback_audit_tags_source_of_truth]):
   plan ファイルの記述と code tag の整合性のみを判定し、tag が「正しい slug を指しているか」
   (= load-bearing hyp が本当に circular か / honest か) の独立 honesty audit は対象外。これは
   `honesty-auditor` の責務 (CLAUDE.md「Independent honesty audit」)。
